## 二、改动清单

### 1. `_roe_g()` 返回值扩展：输出 ROE 原始值

**问题**：`_roe_g()` 内部已算出 ROE，但只返回 `g`，用户看到增长率却无法核查 ROE 来源。

**修复**：返回值改为 `tuple[float, float | None]`，即 `(g, roe)`。

- equity 为零/负时返回 `(0.05, None)`
- 5 个 DCF 函数（`_dcf_capex`、`_dcf_cycle`、`_dcf_std`、`_dcf_pharma`、`_dcf_cycle_soft`）全部改为 `g, roe = _roe_g(...)` 解包，并在 return dict 中加入 `"roe_pct": round(roe * 100, 1)`
- 主流程第964行 `r.get("roe_pct")` 已有，CSV `ROE%` 列对所有 DCF 公司自动填充

### 2. `_roe_g()` 收入萎缩优先逻辑

**问题**：历史收入已在下行时，ROE × 留存收益率仍可能算出正 g，掩盖真实衰退。

**修复**：在计算 g 前先判断收入方向：

```python
g_rev = _revenue_cagr(income_df, cap_hi=999)  # 不钳位上限，只看方向
if g_rev < 0:
    g = float(max(g_rev, cap_lo))  # 收入萎缩：直接用萎缩率
else:
    g = roe * retention            # 收入增长：用 ROE 法
```

收入萎缩时 g 为负值，主流程 `_build_risk_notes` 中的"收入萎缩(x.x%)"标注自动触发。

---

### 3. `_pe_consumer()` 收入萎缩处理

**问题**：消费行业（白酒/啤酒/软饮料/乳制品）一律用 3 年均净利润 × PE，收入下行时均值高估盈利基准；`growth_rate=None` 导致"收入萎缩"标注永不触发。

**修复**：

| 情况 | 盈利基准 | growth_rate |
|------|---------|------------|
| 收入 CAGR ≥ 0 | 3年均净利润（原逻辑） | None |
| 收入 CAGR < 0 | **最近1年净利润** | 收入CAGR（负值，触发标注） |

---

### 4. `_ps_biotech()` 收入萎缩处理

**问题**：`growth_rate=None`，生物制药收入萎缩时无法触发风险标注。

**修复**：

- `_fetch_income` 从取1行改为取3行（至少需2行才能算 CAGR）
- 估值基准仍用最近1年收入（PS 法本来就用最新值，不变）
- 收入 CAGR < 0 时将 CAGR 填入 `growth_rate`，触发"收入萎缩"标注

### 5. 建筑工程行业估值方法调整：pb_roe → dcf_capex

**问题**：建筑工程（中国建筑、中国中铁、中国铁建等）列于 `pb_roe` 组，但该方法假设净资产是定价锚，不适用于以下特征：
- 净资产薄、杠杆率高，PB 锚不稳定

- 债务规模直接影响股权价值

**修复**：`"建筑工程": "pb_roe"` → `"建筑工程": "dcf_capex"`

路桥（招商公路）维持 `pb_roe`：特许经营资产，现金流稳定、监管定价，净资产定价仍适用。

## 七、晚间会话改进（估值模型深化）

### 10. ROE 异常值折扣（`_roe_g()` 内部）

**问题**：ROE 超过 30% 时（高杠杆或单期利润暴增），直接用 `ROE × 留存收益率` 会推出虚高的 g，导致估值失真。

**修复**：超出 30% 的部分打五折，展示原始 ROE 供核查：

```python
if roe > 0.30:
    roe_for_g = 0.30 + (roe - 0.30) * 0.5
else:
    roe_for_g = roe
g = roe_for_g * retention
```

`_build_risk_notes()` 同步新增触发条件：`roe_pct > 30` → 标注”ROE异常偏高(xx.x%)，已折扣处理”。

---

### 11. 情景分析：三个价格区间（`_scenario_fvs()`）

**背景**：单一估值点无法体现 g 假设的敏感性，阅读者无法判断估值区间。

**改动**：

- 新增全局常量 `DELTA_G = 0.03`
- 新增 helper `_scenario_fvs(base_fcf, g, cash, debt, shares, cap_lo, cap_hi, wacc)`，对 g 做 ±3% 扰动并钳位至各方法上下限，返回 `(fv_bear, fv_bull)`
- 5 个 DCF 函数（`_dcf_capex` / `_dcf_cycle` / `_dcf_std` / `_dcf_pharma` / `_dcf_cycle_soft`）各自用本方法的 `cap_lo/cap_hi` 调用并在 return dict 中加入 `fair_bear` / `fair_bull`
- `pb_roe` / `pe_consumer` / `ps_biotech` 不走 EV 路径，填 `None`
- results 新增三列：**`估值_悲观`** / **`估值_中性`** / **`估值_乐观`**

### 12. 行业差异化 WACC（`INDUSTRY_WACC` 字典）

**问题**：统一使用 9% WACC 忽略了行业风险差异——电力/基建资产波动性远低于半导体/创新药。

**改动**：

- 删除硬编码 `WACC = 0.09`，改为 `DEFAULT_WACC = 0.09` + `INDUSTRY_WACC` 字典：

| WACC | 行业 |
|------|------|
| 7% | 火力/水力/新型电力、供气供热、铁路、港口、机场 |
| 8% | 银行、保险、全国地产、区域地产 |
| 9% | 其余行业（默认） |
| 10% | 软件服务、互联网、航空 |
| 11% | 半导体、生物制药 |

- `_dcf_ev(base_fcf, g, wacc)` 加 `wacc` 参数（原用全局常量）
- 所有估值函数签名加 `wacc: float = DEFAULT_WACC`，`_pb_roe` 的戈登公式同步替换
- `UnifiedValuation.calc()` 查字典 `INDUSTRY_WACC.get(industry, DEFAULT_WACC)` 后透传至各函数
