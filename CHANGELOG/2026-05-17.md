### 修复的 Bug（共6处）

**1. `db.py` — 列名未加引号**
- 现象：`macro_shibor` 写入报 `SyntaxError at 'on'`
- 原因：`on`（SQL保留字）和 `1w`（数字开头）未被引号保护
- 修复：upsert 中所有列名统一加双引号

**2. `stock_hist.py` — 相对导入层级错误**
- 现象：`ImportError: cannot import name 'config' from 'fintech_data.tushare'`
- 原因：`from .. import config` 指向了 `tushare` 包，应为 `from ... import config`
- 修复：改为三级相对导入

**3. `lhb.py` — 接口用错**
- 现象：`stk_limit` 返回 `[trade_date, ts_code, up_limit, down_limit]`，无 `exalter` 列
- 原因：`stk_limit` 是涨跌停价格接口，不是龙虎榜营业部接口
- 修复：改用 `top_inst`，同时写入 `stock_top_inst` 和 `stock_lhb_branches_3m`

**4. 龙虎榜表缺唯一约束**
- 现象：`ON CONFLICT` 语句报 `no unique constraint`
- 修复：
  - `stock_top_list`：先去重16行，加主键 `(trade_date, ts_code, reason)`
  - `stock_top_inst`：先去重158行，加主键 `(trade_date, ts_code, exalter)`
  - `stock_lhb_branches_3m`：先去重15,015行，加主键 `(trade_date, ts_code, exalter)`
  - 同步为 `macro_shibor`/`macro_futures`/`macro_global`/`commodity_radar`/`hs300_history`/`index_daily_data` 补加主键

**5. `detector.py` — 查不存在的列**
- 现象：`UndefinedColumn: column "report_year" does not exist`
- 原因：实际表用 `end_date`（Tushare原始字段），旧映射方案中才有 `report_year`
- 修复：改查 `end_date = '20241231'`

**6. 季度财报模块 — 字段映射与实际表结构不符**
- 现象：主键冲突键 `(ts_code, report_year)` 不存在
- 原因：实际主键是 `(ts_code, end_date, report_type)`，且表存储 Tushare 原始字段
- 修复：三个模块全部重写，直接存 Tushare API 原始响应，冲突键改为 `(ts_code, end_date, report_type)`
