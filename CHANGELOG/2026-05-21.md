## 三、新增文件

### `fintech_data/quant/__init__.py`
空模块标记文件。

### `fintech_data/quant/backtest.py`（vectorbt 回测层）

提供从 PostgreSQL 直接取数并用 vectorbt 回测的函数：

| 函数 | 说明 |
|------|------|
| `load_price(symbols, start, end)` | 加载收盘价，返回 date × symbol DataFrame |
| `load_ohlcv(symbol, start, end)` | 加载单股 OHLCV |
| `run_ma_cross(symbol, fast, slow, ...)` | 双均线策略，返回 Portfolio 对象 |
| `run_rsi_strategy(symbol, ...)` | RSI 超买超卖策略 |
| `run_portfolio(symbols, ...)` | 多股等权组合回测 |
| `grid_search_ma(symbol, ...)` | 均线参数网格搜索 |

**技术细节**：用 psycopg2 游标替代 `pd.read_sql`（避免 pandas 2.x 警告），数值列强制 `astype(float)` 以兼容 numba JIT 编译。

### `fintech_data/quant/alpha_factory.py`（qlib 因子层）

预定义 10 个 Alpha 因子表达式（qlib 语法），涵盖动量、均线偏离、波动率、换手率、振幅、RSI 近似、MACD：

```python
ALPHA_FIELDS = {
    "mom_5":     "Ref($close,0)/Ref($close,5)-1",
    "ma5_bias":  "$close/Mean($close,5)-1",
    "vol_20":    "Std($close,20)/$close",
    "rsi_14":    "Mean(If(...),14)/...",
    ...  # 共 10 个
}
```

| 函数 | 说明 |
|------|------|
| `compute_alphas(instruments, fields, start, end)` | 调用 qlib.D.features() 批量计算 |
| `save_alphas_to_pg(df, table)` | 因子结果写回 PostgreSQL（alpha_factors 表）|

**注意**：需先运行 `pg_to_qlib.py` 导出数据后方可使用。

### `fintech_data/quant/pg_to_qlib.py`（qlib 数据导出）

将 `stock_daily_hist` 导出为 qlib CSV provider 格式：

```bash
python -m fintech_data.quant.pg_to_qlib --qlib-dir ~/qlib_data --start 20200101
```

输出结构：
```
~/qlib_data/
├── instruments/all.txt     # 股票列表 + 时间范围
└── csv/<SYMBOL>.csv        # 各股 OHLCV
```

股票代码转换规则：`600519.SH` → `SH600519`（qlib 惯例）。

### `fintech_data/quant/bt_feed.py`（backtrader 接入层）

**PGFeed**：继承 `bt.feeds.PandasData`，一行从 PostgreSQL 加载单股数据：
```python
cerebro.adddata(PGFeed.from_pg("600519.SH", start="2020-01-01"))
```

**内置策略**：

| 策略类 | 参数 | 逻辑 |
|--------|------|------|
| `MACrossStrategy` | fast=10, slow=30 | 金叉买入，死叉卖出 |
| `VolSurgeStrategy` | vol_thresh=2.0, hold_days=5 | 量比>阈值且上涨买入，持N天卖出 |
| `RSIStrategy` | period=14, oversold=30, overbought=70 | RSI 超卖买入，超买卖出 |

**辅助函数**：
- `run_strategy(strategy_cls, symbol, ...)` → 返回统计字典（总收益/夏普/最大回撤/胜率）
- `compare_strategies(symbol, start)` → 四策略并排对比 DataFrame

### `fintech_data/quant/vol_mktcap_analysis.py`（量化分析脚本）

完整的成交量变化 vs 市值变化分析流程：

1. 取沪深300成分股（300只）
2. 加载日线数据（2021至今，约38万行）
3. 计算量比（vol/MA20_vol）、额比（amount/MA20_amount）、前向1/5/20日收益
4. 五分档统计各档均值收益与胜率
5. 皮尔逊相关系数分析（含显著性检验）
6. 茅台单股滞后相关分析
7. vectorbt 量比放量策略回测
